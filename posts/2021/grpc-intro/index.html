Introduction of RPC

RPC(Remote Procedure Call)

What is RPC?

RPC is a network protocol that can call a remote system process like a local system.

  • Client-Server model
  • Procedure call is also known as function call or subrountine call
  • RPC can be implemented throughout HTTP to TCP layers

How does RPC work?

  1. A Client starts a client stub process
  2. Client stub marshels parameters and sends to local RPC runtime
  3. RPC runtime sends to remote RPC runtime
  4. Server side will handle the function call and return the message to client RPC runtime
  5. Client stub unmarshels the return message and returns to the caller

A stub is a piece of code that converts parameters passed between client and server during a remote procedure call.
