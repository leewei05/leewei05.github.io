Go simple HTTP Server

First, create a new directory go-api and main.go

$ mkdir go-api
$ cd go-api
$ touch main.go

Initiate go module

// Make sure GO111MODULE is on
$ export GO111MODULE=on
$ go mod init github.com/leewei05/go-api

Paste the following code in main.go

// main.go
package main

import (
	"database/sql"
	"fmt"
	"log"
	"net/http"
	"time"

	_ "github.com/lib/pq"
)

var (
	db         *sql.DB
	dbHost     = "localhost"
	dbPort     = "5432"
	dbUser     = "postgres"
	dbPwd      = "123456"
	dbName     = "db"
	serverPort = "8080"
)

func initDB() {
	pgStr := fmt.Sprintf("host=%s port=%s user=%s password=%s dbname=%s sslmode=disable",
		dbHost, dbPort, dbUser, dbPwd, dbName)

	if dbHost == "" || dbPort == "" || dbUser == "" || dbPwd == "" || dbName == "" {
		log.Panicf("Missing config parameters: %v", pgStr)
	}

	_, err := sql.Open("postgres", pgStr)
	if err != nil {
		log.Panic("Cannot open PostgreSQL database")
	}
}

func main() {
	initDB()

	port := fmt.Sprintf(":%v", serverPort)

	s := &http.Server{
		Addr:         port,
		ReadTimeout:  10 * time.Second,
		WriteTimeout: 10 * time.Second,
	}

	log.Printf("HTTP server running on port %v", port)
	log.Fatal(s.ListenAndServe())
}

We're going to optimize our code later.

Set up PostgreSQL in Docker

Before we run our go program, we need to set up our local PostgreSQL. I have written a post about How to set up PostgreSQL on Ubuntu 18.04. Personally, I prefer to run Docker container for local testing.

Pull the latest Postgres Docker images

$ Docker pull postgres

Create PostgreSQL local mount point

In order to persist the data that generated by PostgreSQL container, we need to create a local directory to mount to the Docker container. The data running in container will be saved in the data volume we created.

$ mkdir -p ~/docker/volumes/postgres
$ docker run --name pg-docker -e POSTGRES_PASSWORD=123456 -d -p 5432:5432 -v ~/docker/volumes/postgres:/var/lib/postgresql/data  postgres

# Check if container is running as expected
$ docker ps
