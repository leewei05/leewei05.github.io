Leetcode 338: Counting Bits

This question is interesting. I want to write down my solutions and help me review it in the future. The statement of the question is the following:

Question statement

Given a non negative integer number num. For every numbers i in the range 0 ≤ i ≤ num calculate the number of 1’s in their binary representation and return them as an array.

Input: 2 Output: [0,1,1]

Input: 5 Output: [0,1,1,2,1,2]

Solution 1: built-in function

I am using Go to solve this question. The first solution I came up is using the built-in function. Easy peezy.

// Time O(n) + 
// Space O(n)
func countBits(num int) []int {
  res := make([]int,num+1)
  for i:=0;i<=num;i++{
    count := bits.OnesCount(uint(i))
    res[i] = count
  }
  
  return res
}

It worked! Could it be that easy? Since this is a medium level problem. Yeah.. There are some follow ups that we need to consider:

Follow up
  1. It is very easy to come up with a solution with run time O(n*sizeof(integer)). But can you do it in linear time O(n) /possibly in a single pass?
  2. Space complexity should be O(n).
  3. Can you do it like a boss? Do it without using any builtin function like __builtin_popcount in c++ or in any other language.

Solution 2: Not using built-in function

In this solution, we didn’t use the built-in function. Instead, I used a basic bit operation to count the total 1’s in each number.

// Time O(n*sizeof(integer))
// Space O(n)
func countBits(num int) []int {
  res := make([]int,num+1)
  for i:=0;i<=num;i++{
    count:= binaryCount(i)
    res[i] = count
  }
  
  return res
}

func binaryCount(num int)int{
  count := 0
  for num > 0 { 
    count += num & 1
    num >>= 1
  }
  
  return count
}

For instance, 5 is 101 in binary, by using & with 1, we can get the 1’s in each bit. After count += num &1, we use the >> operator to shift right one bit, so we got 10 for our num variable. By repeating this operation, we can get the total 1’s in each number.

However, this solution is conflict with follow up one and two. Damn! Try harder!

Solution 3: Make use of what we have :D

This method is actually posted by someone else, pretty impressive. The first part, try to find a pattern in the following number:

2,4,8,16

No clue? How about now?

10000
01000
00100
00010

We can see that even numbers don’t have 1 in the first bit. Therefore, we can skip the first bit for even numbers. Let’s take 6 for example, 6’s binary is 110. 6 and 3(11) has the same amount of 1’s. So, we have the conclusion for even numbers: res[6]=res[3] (6 >> 1 = 3)

As for odd numbers, it is pretty straght forward. Odd numbers will always have 1 in the first bit. And we can get the total amount of 1’s based on the previous number.

For instance, 5 is 101, 4 is 100. 5 is 100+1, we can get the conclusion for odd numbers: res[5]=res[4]+1 By turning the actual numbers into variable i, we can get the following solution.

func countBits(num int) []int {
  res := make([]int,num+1)
  for i:=0;i<=num;i++{
    if i&1 == 0{
      res[i]=res[i>>1]
    }else{
      res[i]=res[i-1]+1
    }
  }
  
  return res
}

We didn’t use built-in function. Our time and space complexity are O(n). Nicely done!
